# DISC16 행동스타일 코칭검사 QA 체크리스트

## 1. 검사 시작 화면

- [ ] 제목과 설명이 360px 폭에서 잘리지 않는다.
- [ ] CTA가 한 화면에서 식별된다.
- [ ] MBTI 비공식 혼동 방지 문구가 표시된다.
- [ ] 검사 목적이 진단·채용판정이 아닌 자기이해와 코칭대화로 안내된다.

## 2. 문항 진행

- [ ] 첫 문항 진행률이 5%로 표시된다.
- [ ] 문항 번호가 1/20부터 20/20까지 정확히 표시된다.
- [ ] 각 문항에 D·I·S·C 문장이 한 개씩 포함된다.
- [ ] 문장 위치가 문항마다 분산되어 유형과 위치가 고정되지 않는다.
- [ ] 첫 선택에 1, 두 번째 선택에 2, 세 번째 선택에 3, 네 번째 선택에 4가 표시된다.
- [ ] 선택 순위는 색뿐 아니라 숫자와 테두리로 구분된다.
- [ ] 선택한 중간 순위를 누르면 그 순위 이후 선택이 해제된다.
- [ ] 네 번째 선택 후 자동으로 다음 문항으로 이동한다.
- [ ] 자동 이동 직전 재선택하면 예약 이동이 취소된다.
- [ ] 이전 문항 이동 시 예약된 자동 이동이 취소된다.
- [ ] 문항 초기화 시 예약된 자동 이동이 취소된다.
- [ ] 이전 문항으로 돌아가면 기존 순위가 유지된다.
- [ ] 새로고침 후 현재까지의 응답이 유지된다.

## 3. 채점 검증

순위 점수:

- 1순위: 4점
- 2순위: 3점
- 3순위: 2점
- 4순위: 1점

검증 항목:

- [ ] 모든 문항에서 같은 유형을 1순위로 선택하면 해당 유형이 주성향이 된다.
- [ ] 각 유형 원점수 범위가 20~80점 안에 있다.
- [ ] 각 유형 10척도 값이 0.0~10.0 안에 있다.
- [ ] 네 유형 원점수 합이 항상 200점이다.
- [ ] 동점일 때 브라우저별 결과 순서가 달라지지 않는지 확인한다.
- [ ] 1위와 2위 차이가 10점 이상이면 단일형으로 표시된다.
- [ ] 1위와 2위 차이가 10점 미만이면 복합형으로 표시된다.

## 4. 좌표 검증

좌표 공식:

- X축 = D+I 대 S+C
- Y축 = D+C 대 I+S

분면:

- 오른쪽 위: D
- 오른쪽 아래: I
- 왼쪽 아래: S
- 왼쪽 위: C

검증 항목:

- [ ] D 편향 응답은 오른쪽 위에 표시된다.
- [ ] I 편향 응답은 오른쪽 아래에 표시된다.
- [ ] S 편향 응답은 왼쪽 아래에 표시된다.
- [ ] C 편향 응답은 왼쪽 위에 표시된다.
- [ ] 좌표 값이 0.0~10.0 안에 있다.
- [ ] 주성향과 보조성향 색이 좌표점에 함께 표시된다.
- [ ] 넓은 분포가 좁은 분포보다 크게 표시된다.
- [ ] 보조성향 방향으로 타원이 확장된다.
- [ ] 축 설명이 모바일에서도 잘리지 않는다.

## 5. 16유형 결과

단일형:

- [ ] D 돌파하는 추진가
- [ ] I 사람을 여는 연결자
- [ ] S 곁을 지키는 조율자
- [ ] C 기준을 세우는 설계자

복합형:

- [ ] DI 사람을 움직이는 추진가
- [ ] DS 책임지는 실행리더
- [ ] DC 전략을 세우는 승부사
- [ ] ID 도전을 확산하는 선도자
- [ ] IS 마음을 모으는 활력가
- [ ] IC 아이디어를 다듬는 기획자
- [ ] SD 조용히 이끄는 책임자
- [ ] SI 사람을 품는 조율자
- [ ] SC 빈틈을 지키는 운영자
- [ ] CD 답을 찾아내는 전략가
- [ ] CI 생각을 전하는 해석자
- [ ] CS 질서를 지키는 관리자

각 유형 공통 확인:

- [ ] 유형명과 주성향·보조성향이 일치한다.
- [ ] 대표 관계 장면이 유형에 맞게 달라진다.
- [ ] 좋은 의도와 상대 경험이 구분된다.
- [ ] 과도사용을 결함이나 부정적 낙인으로 표현하지 않는다.
- [ ] 관계실험이 구체적인 행동으로 제시된다.
- [ ] 코칭질문 네 개가 표시된다.

## 6. 디자인·접근성

- [ ] 본문과 배경의 명도 대비가 충분하다.
- [ ] 색을 구분하지 못해도 유형 문자와 숫자로 내용을 이해할 수 있다.
- [ ] D 색과 오류·경고 색이 혼동되지 않는다.
- [ ] 골드가 유형 우열이 아닌 선택·성찰·완료에만 사용된다.
- [ ] 360px, 390px, 412px 모바일 폭에서 가로 스크롤이 없다.
- [ ] 200% 확대에서도 주요 기능을 사용할 수 있다.
- [ ] 터치 영역이 지나치게 작지 않다.
- [ ] 긴 문항도 순위 숫자와 겹치지 않는다.

## 7. 저장·재검사

- [ ] 검사 도중 이탈 후 다시 들어오면 응답이 유지된다.
- [ ] 완료된 응답이 있으면 저장된 결과 진입 버튼이 표시된다.
- [ ] 다시 검사하기를 누르면 저장 응답이 삭제된다.
- [ ] 다시 검사하기 후 첫 화면으로 정상 복귀한다.
- [ ] 잘못된 localStorage 데이터가 있을 때 페이지가 완전히 멈추지 않는지 확인한다.

## 8. 출시 전 운영 기준

- [ ] 최소 20명 예비검사를 진행한다.
- [ ] 문항별 1순위 선택률을 확인한다.
- [ ] 특정 유형 문장이 과도하게 매력적인지 확인한다.
- [ ] 유형별 결과 분포를 확인한다.
- [ ] 1·2위 점수차 분포를 확인해 단일형 기준을 조정한다.
- [ ] 사용자에게 결과가 자신의 실제 관계 장면과 연결되는지 질문한다.
- [ ] 사용자에게 불편하거나 낙인처럼 느껴지는 표현이 있는지 확인한다.
- [ ] 검사 결과가 다른 사람 평가나 채용판정에 사용되지 않도록 안내한다.

- 결과 인쇄 및 PDF 저장 최적화

- 두 사람의 유형 비교
- 유형별 코칭질문 확장
- 응답 데이터 기반 문항 분석
