# DISC16 행동스타일 코칭검사 QA 체크리스트 v2

## 1. 검사 시작 화면

- [ ] 제목과 설명이 360px 폭에서 잘리지 않는다.
- [ ] CTA가 한 화면에서 식별된다.
- [ ] 16개 상황문항과 1·2·3·4순위 선택 방식이 안내된다.
- [ ] 결과에서 1·2·3순위 종합분석을 제공한다는 설명이 표시된다.
- [ ] MBTI 비공식 혼동 방지 문구가 표시된다.
- [ ] 검사 목적이 진단·채용판정이 아닌 자기이해와 코칭대화로 안내된다.

## 2. 상황지문과 키워드 보기

- [ ] 문항 번호가 1/16부터 16/16까지 정확히 표시된다.
- [ ] 첫 문항 진행률이 6.25%로 표시된다.
- [ ] 각 문항은 실제 관계·업무 장면을 충분히 설명한다.
- [ ] 지문만 읽어도 사용자가 자신의 경험을 떠올릴 수 있다.
- [ ] 보기는 긴 문장 대신 짧은 행동 키워드로 표시된다.
- [ ] 각 문항에 D·I·S·C 키워드가 한 개씩 포함된다.
- [ ] 키워드 위치가 문항마다 분산되어 유형과 화면 위치가 고정되지 않는다.
- [ ] 키워드가 지나치게 긍정적이거나 부정적으로 보이지 않는다.
- [ ] 특정 유형 키워드가 다른 키워드보다 항상 길거나 전문적으로 보이지 않는다.
- [ ] 모바일에서 긴 지문과 키워드 카드가 겹치거나 잘리지 않는다.

## 3. 순위선택 동작

- [ ] 첫 선택에 1, 두 번째 선택에 2, 세 번째 선택에 3, 네 번째 선택에 4가 표시된다.
- [ ] 선택 순위는 색뿐 아니라 숫자와 테두리로 구분된다.
- [ ] 선택한 중간 순위를 누르면 해당 순위 이후 선택이 해제된다.
- [ ] 네 번째 선택 후 자동으로 다음 문항으로 이동한다.
- [ ] 자동 이동 직전 재선택하면 예약 이동이 취소된다.
- [ ] 이전 문항 이동 시 예약된 자동 이동이 취소된다.
- [ ] 문항 초기화 시 예약된 자동 이동이 취소된다.
- [ ] 이전 문항으로 돌아가면 기존 순위가 유지된다.
- [ ] 새로고침 후 현재까지의 응답이 유지된다.

## 4. 채점 검증

순위 점수:

- 1순위: 4점
- 2순위: 3점
- 3순위: 2점
- 4순위: 1점

검증 항목:

- [ ] 모든 문항에서 같은 유형을 1순위로 선택하면 해당 유형이 주성향이 된다.
- [ ] 각 유형 원점수 범위가 16~64점 안에 있다.
- [ ] 각 유형 10척도 값이 0.0~10.0 안에 있다.
- [ ] 네 유형 원점수 합이 항상 160점이다.
- [ ] 동점일 때 D·I·S·C 고정순서로 결과가 안정적으로 정렬된다.
- [ ] 1위와 2위 차이가 8점 이상이면 단일형 코드로 표시된다.
- [ ] 1위와 2위 차이가 8점 미만이면 복합형 코드로 표시된다.
- [ ] 1·2·3·4순위가 결과 전 구간에서 동일하게 유지된다.

## 5. 좌표 검증

좌표 공식:

- X축 = D+I 대 S+C
- Y축 = D+C 대 I+S

분면:

- 오른쪽 위: D
- 오른쪽 아래: I
- 왼쪽 아래: S
- 왼쪽 위: C

검증 항목:

- [ ] D 편향 응답은 오른쪽 위에 표시된다.
- [ ] I 편향 응답은 오른쪽 아래에 표시된다.
- [ ] S 편향 응답은 왼쪽 아래에 표시된다.
- [ ] C 편향 응답은 왼쪽 위에 표시된다.
- [ ] 좌표 값이 0.0~10.0 안에 있다.
- [ ] 1·2·3순위 색이 좌표점에 함께 표시된다.
- [ ] 넓은 분포가 좁은 분포보다 크게 표시된다.
- [ ] 축 설명이 모바일에서도 잘리지 않는다.

## 6. 1·2·3순위 종합분석

- [ ] 1순위는 행동의 출발점으로 설명된다.
- [ ] 2순위는 주성향을 보완하거나 조절하는 방식으로 설명된다.
- [ ] 3순위는 필요할 때 사용하는 조절 자원으로 설명된다.
- [ ] 4순위는 약점이나 결함이 아니라 의식적으로 요청·연습할 성향으로 설명된다.
- [ ] 1·2위 조합에 맞는 복합유형명이 표시된다.
- [ ] 회의·협업, 가까운 관계, 압박 장면 해석이 조합별로 달라진다.
- [ ] 상위 세 성향 설명이 단순 반복되지 않고 역할 차이가 드러난다.
- [ ] 종합분석을 읽었을 때 사용자가 자신의 실제 관계 장면을 떠올릴 수 있다.

## 7. 네 성격 상세해석

각 성향 카드에 다음 내용이 모두 표시되는지 확인한다.

- [ ] 순위와 10척도 점수
- [ ] 기본 특징
- [ ] 잘 작동할 때의 강점
- [ ] 압박받을 때의 과도사용
- [ ] 대인관계에서 주변이 경험할 수 있는 모습
- [ ] 편안한 교류 조건
- [ ] 성장 방향

표현 원칙:

- [ ] D를 공격적·독단적인 사람으로 단정하지 않는다.
- [ ] I를 가볍거나 책임감 없는 사람으로 단정하지 않는다.
- [ ] S를 소극적이거나 우유부단한 사람으로 단정하지 않는다.
- [ ] C를 차갑거나 비판적인 사람으로 단정하지 않는다.
- [ ] 모든 성향에서 강점 → 과도사용 → 성장선택의 구조가 유지된다.

## 8. 다른 성향과의 대인관계 교류

- [ ] D·I·S·C 상대에게 필요한 대화방식이 각각 제공된다.
- [ ] 사용자의 1순위 성향에 따라 주의할 교류방식이 달라진다.
- [ ] 같은 유형끼리 만났을 때 발생할 수 있는 과도사용이 설명된다.
- [ ] 다른 유형과의 차이를 우열이 아니라 속도·관심·기준 차이로 설명한다.
- [ ] 관계 팁이 바로 사용할 수 있는 행동문장으로 제시된다.

## 9. 16유형 결과명

단일형:

- [ ] D 주도형
- [ ] I 영향형
- [ ] S 안정형
- [ ] C 신중형

복합형:

- [ ] DI 사람을 움직이는 추진가
- [ ] DS 책임지는 실행리더
- [ ] DC 전략을 세우는 승부사
- [ ] ID 도전을 확산하는 선도자
- [ ] IS 마음을 모으는 활력가
- [ ] IC 아이디어를 다듬는 기획자
- [ ] SD 조용히 이끄는 책임자
- [ ] SI 사람을 품는 조율자
- [ ] SC 빈틈을 지키는 운영자
- [ ] CD 답을 찾아내는 전략가
- [ ] CI 생각을 전하는 해석자
- [ ] CS 질서를 지키는 관리자

## 10. 디자인·접근성

- [ ] 본문과 배경의 명도 대비가 충분하다.
- [ ] 색을 구분하지 못해도 유형 문자·순위·숫자로 내용을 이해할 수 있다.
- [ ] D 색과 오류·경고 색이 혼동되지 않는다.
- [ ] 골드가 유형 우열이 아닌 선택·성찰·완료에만 사용된다.
- [ ] 360px, 390px, 412px 모바일 폭에서 가로 스크롤이 없다.
- [ ] 200% 확대에서도 주요 기능을 사용할 수 있다.
- [ ] 터치 영역이 지나치게 작지 않다.
- [ ] 긴 지문이 보기 선택 영역을 지나치게 아래로 밀지 않는다.
- [ ] 결과의 상세 카드가 모바일에서 한 열로 자연스럽게 정렬된다.

## 11. 저장·재검사

- [ ] v2 저장키 `disc16:answers:v2`가 사용된다.
- [ ] 기존 v1 응답이 v2 결과에 잘못 섞이지 않는다.
- [ ] 검사 도중 이탈 후 다시 들어오면 v2 응답이 유지된다.
- [ ] 완료된 응답이 있으면 저장된 결과 진입 버튼이 표시된다.
- [ ] 다시 검사하기를 누르면 v2 저장 응답이 삭제된다.
- [ ] 다시 검사하기 후 첫 화면으로 정상 복귀한다.
- [ ] 잘못된 localStorage 데이터가 있을 때 페이지가 완전히 멈추지 않는다.

## 12. 출시 전 운영 기준

- [ ] 최소 30명 예비검사를 진행한다.
- [ ] 문항별 1순위 선택률을 확인한다.
- [ ] 문항별 네 키워드 선택순위 분포를 확인한다.
- [ ] 특정 키워드가 과도하게 매력적인지 확인한다.
- [ ] 유형별 결과 분포를 확인한다.
- [ ] 1·2위 점수차 분포를 확인해 단일형 기준 8점을 재검토한다.
- [ ] 상위 3순위 해석이 실제 자신과 맞는지 5점 척도로 평가받는다.
- [ ] 관계 장면 해석이 구체적으로 이해되는지 사용자 의견을 받는다.
- [ ] 불편하거나 낙인처럼 느껴지는 표현이 있는지 확인한다.
- [ ] 검사 결과가 타인 평가나 채용판정에 사용되지 않도록 안내한다.
